# HTB Starting Point — Crocodile

## Resumo

Neste lab, foi feita enumeração de serviços com Nmap, acesso anônimo via FTP, coleta de credenciais e autenticação em uma página web encontrada com Gobuster.

## Alvo

```text
IP: 10.129.98.190
Serviços principais:
21/tcp  FTP
80/tcp  HTTP
```

## 1. Enumeração com Nmap

```bash
nmap -sC -sV 10.129.98.190
```

Parâmetros:

```text
-sC = executa scripts padrão do Nmap
-sV = detecta serviço e versão
```

O scan identificou FTP e HTTP abertos.

No HTTP, foi identificado:

```text
Apache httpd 2.4.41
```

## 2. Acesso FTP anônimo

O FTP permitia login anônimo.

```bash
ftp 10.129.98.190
```

Usuário:

```text
anonymous
```

Código retornado:

```text
230
```

Esse código indica login bem-sucedido.

## 3. Download dos arquivos

Dentro do FTP, foram listados os arquivos:

```ftp
ls
```

Para baixar:

```ftp
get nome_do_arquivo
```

Os arquivos baixados continham listas de usuários e senhas.

## 4. Enumeração web com Gobuster

Como a porta 80 estava aberta, foi feita enumeração de diretórios e arquivos PHP:

```bash
gobuster dir -u http://10.129.98.190 -w /usr/share/wordlists/dirb/common.txt -x php
```

Parâmetros:

```text
dir = modo de busca por diretórios/arquivos
-u   = URL alvo
-w   = wordlist
-x   = extensões de arquivo
```

O arquivo encontrado foi:

```text
login.php
```

## 5. Login no painel

Com as credenciais encontradas no FTP, foi acessado:

```text
http://10.129.98.190/login.php
```

Após autenticar, a flag foi exibida na página.

## Conceitos aprendidos

```text
FTP anônimo pode expor arquivos sensíveis.
Arquivos como userlist e passwd podem conter credenciais.
HTTP aberto deve ser analisado no navegador e com Gobuster.
Gobuster com -x php ajuda a encontrar páginas como login.php.
Credenciais encontradas em um serviço podem ser reutilizadas em outro.
```

## Fluxo final

```text
Nmap → FTP anônimo → baixar arquivos → encontrar credenciais → Gobuster → login.php → autenticar → flag
```

## Conclusão

O lab mostrou uma falha simples de exposição de arquivos via FTP anônimo. As credenciais obtidas no FTP foram reutilizadas no serviço web, permitindo acesso ao painel e captura da flag.
