# Hack The Box — Fawn

## Objetivo

Explorar o serviço FTP exposto na máquina alvo e encontrar a flag armazenada no servidor.

## O que aprendi

Neste lab, aprendi conceitos básicos sobre FTP (File Transfer Protocol), um protocolo utilizado para transferência de arquivos entre clientes e servidores.

Também aprendi:

* Como identificar um serviço FTP com Nmap.
* O conceito de login anônimo (Anonymous FTP).
* Como navegar em um servidor FTP.
* Como baixar arquivos utilizando o comando `get`.

## Reconhecimento

Primeiro realizei um scan utilizando o Nmap para identificar quais serviços estavam expostos:

```bash
nmap -sV -sC IP
```

O resultado indicou que a porta 21 estava aberta e executando o serviço FTP.

```text
21/tcp open ftp
```

## Enumeração

Ao pesquisar sobre FTP, descobri que muitos servidores permitem acesso utilizando uma conta padrão chamada:

```text
anonymous
```

Esse tipo de acesso é conhecido como Anonymous FTP, permitindo que usuários acessem arquivos públicos sem possuir uma conta cadastrada.

Conectei ao serviço utilizando:

```bash
ftp IP
```

Login:

```text
Username: anonymous
Password: anonymous
```

O acesso foi concedido com sucesso.

## Exploração

Após o login, listei os arquivos disponíveis no servidor:

```bash
ls
```

Foi possível visualizar o arquivo contendo a flag.

## Captura da Flag

Como eu estava conectado apenas ao serviço FTP e não ao sistema operacional da máquina, não era possível utilizar comandos Linux como:

```bash
cat arquivo.txt
```

Isso acontece porque o FTP fornece acesso aos arquivos disponibilizados pelo servidor, mas não fornece um terminal remoto da máquina.

Por esse motivo foi necessário utilizar o comando:

```bash
get nome_do_arquivo
```

O comando `get` faz o download do arquivo do servidor FTP para minha máquina local.

Após baixar o arquivo, utilizei:

```bash
cat nome_do_arquivo
```

para visualizar seu conteúdo e obter a flag.

## Por que usamos o comando GET?

Durante o lab não obtivemos acesso à máquina do alvo, como aconteceu no lab Meow através do Telnet.

No Fawn tivemos acesso apenas ao serviço FTP, que funciona como um servidor de arquivos.

Por isso o arquivo precisou ser transferido do servidor para nossa máquina utilizando:

```bash
get arquivo.txt
```

Somente depois do download foi possível abrir e ler o conteúdo localmente.

## Comandos utilizados

```bash
nmap -sV -sC IP
ftp IP
ls
get arquivo.txt
cat arquivo.txt
```
